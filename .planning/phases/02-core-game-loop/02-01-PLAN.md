Create `src/lib/queries/leaderboard.ts`:

1. Export `LeaderboardRow` interface: `{ userId: number, name: string, teamId: number | null, teamSlug: string | null, totalXp: number, todayXp: number, rank: number, prevRank: number | null }`.

2. Export `getLeaderboard(opts: { period: 'today' | 'week' | 'month' | 'all', teamId: number | null })` that returns `Promise<LeaderboardRow[]>`.

3. Use `db.execute(sql`...`)` with a CTE approach:
   - `period_xp` CTE: JOIN `users` (WHERE role='rep') LEFT JOIN `activities` with period filter (date_trunc for day/week/month on `created_at AT TIME ZONE 'UTC'`, no filter for 'all'). Add team filter if `teamId` is not null. SELECT userId, name, teamId, SUM(xp_earned) as totalXp, SUM(CASE WHEN created_at >= date_trunc('day', NOW() AT TIME ZONE 'UTC') THEN xp_earned ELSE 0 END) as todayXp. GROUP BY user columns.
   - Also JOIN `teams` to get `teamSlug` (needed for team filter UI).
   - Main SELECT: `RANK() OVER (ORDER BY "totalXp" DESC)::int AS rank` from the CTE.
   - Cast numeric results with `::int` to avoid string return from Postgres.

4. For rank movement (LEAD-06): Export `getPreviousRanks(opts: { period, teamId })` that runs the same query but with the PREVIOUS period's date range. For 'today' -> yesterday's full day. For 'week' -> last week. For 'month' -> last month. For 'all' -> return null (no previous). Returns `Map<number, number>` (userId -> previousRank). This is computed in application code by comparing the two result sets.

5. Export `getGapMessage(currentXP: number, nextRankXP: number, nextRankPosition: number): string` helper. Calculate gap = nextRankXP - currentXP + 1. Express in calls if <= 5 calls needed, meetings if <= 3 meetings needed, else raw XP. Use XP_VALUES from constants.

6. Export `getTier(xp: number): 'gold' | 'silver' | 'bronze'` helper using TIER_THRESHOLDS.

Create `src/lib/queries/streaks.ts`:

1. Export `StreakResult` interface: `{ userId: number, currentStreak: number, streakActive: boolean }`.

2. Export `getStreaks(): Promise<StreakResult[]>` using gap-and-island SQL technique from RESEARCH.md. Key points:
   - Filter to business days only: `EXTRACT(DOW FROM created_at) NOT IN (0, 6)`
   - Count calls per user per business day
   - Only qualifying days with >= ACTIVITY_STREAK_THRESHOLD calls
   - Gap-and-island: subtract row_number from activity_date to find consecutive groups
   - Return most recent streak per user with DISTINCT ON
   - `streakActive` is true if last qualifying day is today or the most recent business day (Friday if today is weekend, yesterday if weekday)

Create `src/lib/queries/funnel.ts`:

1. Export `FunnelResult` interface: `{ userId: number, calls: number, meetings: number, demos: number, deals: number }`.

2. Export `getFunnel(userId?: number): Promise<FunnelResult[]>`. COUNT activities grouped by type per user (all time). If userId provided, filter to that user. Use standard Drizzle query with `db.select()` and `sql` for the conditional counting: `COUNT(*) FILTER (WHERE type = 'call')` etc.
